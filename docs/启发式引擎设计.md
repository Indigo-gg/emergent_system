# 启发式引擎设计文档

## 一、设计哲学

### 1.1 核心原则：不要模拟行为，模拟物理

传统做法是用 `if-else` 堆砌高级行为规则：
```
if 营养 < 10: 休眠()
if 相遇: 弹开()
if 能量 > 阈值: 分裂()
```

这种"繁杂系统"（Complicated）像波音飞机——零件极多，少一个就飞不起来，很难产生意料之外的涌现。

本引擎的设计目标是"复杂系统"（Complex）——像水或鸟群，规则极少，但因非线性交互涌现出千变万化的形态。

**底层规则不包含"目的性"**，全部是对能量、距离、化学梯度和相位的纯粹数学运算。

### 1.2 从启发方案到引擎的映射

| 启发方案维度 | 对应物理量 | 引擎模块 | 计算复杂度 |
|-------------|-----------|---------|-----------|
| 能量代谢与"懒惰" | 能量守恒 | `environment.py` 粒子交互 | O(N) |
| 物质扩散场（图灵斑图） | 化学浓度 | `environment.py` 反应扩散 | O(W×H) |
| 引斥力与柔性结构 | 距离势能 | `lj_potential.py` | O(N×邻居) |
| 内部节律与"心跳" | 相位振荡 | `kuramoto.py` | O(N×邻居) |
| 属性向量与生态位 | 博弈内积 | `step.py` trait 调制 | O(N×邻居) |

### 1.3 尺度不变性

粒子不是具体的生命，而是"交流隔绝的抽象实体"——可以是一个细胞、一个人、一个国家。正因交流隔绝，它们才能维持简单状态，涌现只需调整内部特性和外部环境。

这意味着同一套规则可以解释：
- 细胞形成组织
- 人群形成部落
- 国家形成联盟

## 二、五维机制详解

### 2.1 能量代谢（对抗熵增的本能）

**直觉**：在资源匮乏的环境里，乱动会死。

**底层规则**：
- 每次移动消耗能量：`代谢 = 基础代谢 + 移动系数 × 速度`
- 废弃物区域增加代谢惩罚：`代谢 × (1 + 废弃物浓度 × 惩罚系数)`
- 能量 ≤ 0 → 进入休眠（极低代谢，速度归零）
- 休眠期间附近有营养 → 醒来
- 休眠过久 → 永久死亡

**涌现效果**：
- 粒子自发降低活跃度（宏观上的"懒惰"）
- 资源丰富时扩张，匮乏时收缩
- 周期性的崩溃与复苏

**关键参数**：
```yaml
environment:
  base_metabolism: 0.01      # 静止时每步消耗
  move_cost: 0.005           # 速度每单位额外消耗
  absorb_rate: 0.5           # 营养吸收效率
  dormant_metabolism: 0.001  # 休眠时极低消耗
  max_dormant_ticks: 600     # 休眠多久后死亡
```

**设计要点**：不要用 `if 能量<阈值: 休眠()`，而是让"速度与能量成正比"——这是一条连续函数，休眠是它的自然极限情况。

### 2.2 多通道反应扩散（图灵斑图）

**直觉**：环境不只是容器，它是两种物质博弈的温床。

**底层规则（Gray-Scott 模型）**：
```
dA/dt = Da × ∇²A - A×B² + f×(1-A)
dB/dt = Db × ∇²B + A×B² - (k+f)×B
```

- A：慢扩散促进剂（信号物质，粒子可以沉积）
- B：快扩散抑制剂
- A 促进 B 的生成，B 消耗 A
- 扩散速率不同 → 自发形成斑点或条纹

**涌现效果**：
- 即使没有粒子，环境自身也会形成稳定的图灵斑图
- 粒子在其中运动时，沿着斑图边界聚集
- 类似"血管网"或"龟甲"的结构

**关键参数**：
```yaml
reaction_diffusion:
  da: 0.16      # A 扩散率（慢）
  db: 0.08      # B 扩散率
  feed: 0.055   # 补给率（控制斑图密度）
  kill: 0.062   # 消除率（与 feed 配合决定斑图类型）
```

**调参指南**：Gray-Scott 参数空间极其敏感：
- `feed=0.055, kill=0.062` → 经典斑点
- `feed=0.037, kill=0.06` → 条纹
- `feed=0.025, kill=0.055` → 螺旋波
- 微小的参数变化会导致完全不同的斑图形态

### 2.3 Lennard-Jones 连续引斥力（柔性结构）

**直觉**：靠得太近会抢资源（排斥），离太远容易被环境淘汰（吸引）。

**底层规则**：
```
U(r) = 4ε [(σ/r)¹² - (σ/r)⁶]
F(r) = 24ε [2(σ/r)¹² - (σ/r)⁶] / r
```

- r = σ：力为零（平衡距离）
- r < σ：强烈斥力（防止重叠）
- r > σ：引力（抱团取暖）
- r > cutoff：无力（节省计算）

**涌现效果**：
- 粒子群自发维持间距，形成类似细胞膜的柔性边界
- 受外力时整体形变而非散开
- 表现出"表面张力"效果

**关键参数**：
```yaml
lj:
  sigma: 0.5        # 平衡距离 ≈ 粒子半径
  epsilon: 1.0      # 势阱深度（"黏性"）
  cutoff: 2.5       # 截断距离 = 5σ
  max_force: 50.0   # 力上限（L-J 近距离力很大）
```

**与硬碰撞的区别**：
- 旧方案：`if 距离 < 阈值: 弹开()` → 硬边界，无柔性
- 新方案：连续力场 → 柔性结构，距离越近斥力越强但不是突变

### 2.4 Kuramoto 相位同步（集体"心跳"）

**直觉**：生命都有内在节奏——日出日落，心脏跳动。

**底层规则**：
```
dθ_i/dt = ω_i + (K/N) Σ_j sin(θ_j - θ_i)
```

- θ：当前相位 [0, 2π]
- ω：自然频率（每个粒子不同，产生个体差异）
- K：耦合强度（越大同步越快）
- 相邻粒子会微调彼此的相位

**涌现效果**：
- 粒子群自发同步相位，形成周期性聚集-扩散（"心跳"）
- 不同 K 值下出现：部分同步、完全同步、混沌
- 相位波峰时全速运动，波谷时半速 → 集体"呼吸"

**速度调制**：
```python
phase_mod = 0.5 + 0.5 * sin(phase)   # [0, 1]
effective_speed = max_speed * phase_mod
```

**关键参数**：
```yaml
kuramoto:
  coupling_strength: 2.0   # K 值，越大同步越快
```

**与 L-J 力的组合效果**：L-J 力提供空间结构，Kuramoto 提供时间节律 → 粒子群像一团有节奏搏动的活组织。

### 2.5 属性向量与生态位分化（零和博弈的出口）

**直觉**：如果所有粒子都同质，系统很快陷入死寂内耗。

**底层规则**：
- 每个粒子持有一个 3 维属性向量 `trait = [攻击, 采集, 合作]`
- 交互时计算 trait 内积（相似度）：`sim = dot(trait_i, trait_j)`
- 相似 → 增强引力（抱团），差异 → 增强斥力（领地）
- 繁殖时 trait 加微小随机变异

**涌现效果**：
- 自发分化出"掠夺者"（高攻击、低合作）和"采集者"（高采集、高合作）
- 不同 trait 的粒子占据不同空间区域，形成领地边界
- 系统从单一抢夺演化为相互依存的生态链

**力的调制方式**：
```python
similarity = dot(trait_i, trait_j)       # [0, 3]
sim_norm = similarity / 1.5 - 1.0       # [-1, 1]
trait_factor = 1.0 + sim_norm × strength # [0.5, 1.5]
F_final = F_gep × trait_factor
```

**关键参数**：
```yaml
trait:
  dim: 3                    # [攻击, 采集, 合作]
  influence_strength: 0.5   # 调制幅度
  mutation_sigma: 0.05      # 变异幅度
```

## 三、力架构总览

### v7 力合成公式

```
F_total = F_lj + F_gep × trait_factor + F_env
```

| 分量 | 来源 | 作用 |
|------|------|------|
| F_lj | Lennard-Jones 势 | 近斥远引，形成柔性结构 |
| F_gep | GEP 演化的势函数 | 粒子间环境感知的交互力 |
| trait_factor | trait 内积 | 调制 F_gep，相似吸引差异排斥 |
| F_env | GEP 演化的趋化公式 | 沿环境梯度的趋向力 |

### 每步执行流程

```
1. 构建空间哈希（GPU 直传，无 CPU 往返）
2. 环境步进
   ├── 营养扩散 + 衰减
   ├── 废弃物扩散 + 衰减
   └── 反应扩散（Gray-Scott）
3. 计算梯度（所有 4 个通道）
4. 计算力（F_lj + F_gep × trait + F_env）
5. 积分（含相位速度调制）
6. Kuramoto 相位同步
7. 粒子-环境交互（吸收、排泄、代谢、休眠、死亡）
```

## 四、工程优化

### 4.1 消除 CPU-GPU 往返

**问题**：每步将 50 万粒子坐标拉回 CPU numpy 再传给空间哈希，50000 步 = 200GB 无效传输。

**方案**：`spatial_hash.build()` 参数从 `ti.types.ndarray()` 改为 `ti.template()`，直接接收 GPU 上的 Taichi field。

### 4.2 全张量化

```
消除一切循环，拥抱矩阵
消除一切对象，拥抱张量
```

系统中只存在几个巨大的 Taichi field：
- `pos_x/y, vel_x/y`: (N,) 坐标和速度
- `force_x/y`: (N,) 力累加器
- `energy, phase, dphase`: (N,) 内部状态
- `trait`: (N, 3) 属性向量
- `nutrient/waste/signal_a/signal_b`: (W, H) 环境场

所有规则迭代都是对这些 field 的纯粹数学运算。

### 4.3 计算复杂度

| 操作 | 复杂度 | 8G 显卡承载量 |
|------|--------|-------------|
| 能量代谢 | O(N) | 500万+ 粒子 |
| 反应扩散 | O(W×H) | 1024×1024 网格 |
| 邻居力计算 | O(N×邻居) | 50万粒子（空间哈希后） |
| Kuramoto | O(N×邻居) | 同上 |
| 梯度计算 | O(W×H) | 同环境网格 |

## 五、配置指南

### 5.1 新机制开关

所有新机制都可以通过配置独立开关：

```yaml
lj:
  enabled: true          # Lennard-Jones 引斥力
reaction_diffusion:
  enabled: true          # 反应扩散
kuramoto:
  enabled: true          # 相位同步
trait:
  enabled: true          # 属性向量
```

关闭任意一个会回退到旧行为（向后兼容）。

### 5.2 推荐调参顺序

1. **先跑基线**：关闭所有新机制，确认旧逻辑正常
2. **开 L-J**：调 `sigma` 和 `epsilon`，观察柔性结构
3. **开反应扩散**：调 `feed` 和 `kill`，观察斑图
4. **开 Kuramoto**：调 `coupling_strength`，观察同步
5. **开 trait**：调 `influence_strength`，观察分化

### 5.3 快速调试配置

```yaml
# fast_debug.yaml — 5000 粒子，快速迭代
simulation:
  num_particles: 5000
  steps_per_eval: 1000
```

## 六、预期涌现现象

| 机制组合 | 预期涌现 |
|---------|---------|
| 能量 + L-J | 柔性"细胞膜"，受力形变而非散开 |
| 反应扩散 + 趋化 | 粒子沿图灵斑图边界聚集，形成血管网 |
| L-J + Kuramoto | 有节奏搏动的活组织 |
| 全部组合 | 自发分化的生态系统：掠夺者、采集者、共生者 |
